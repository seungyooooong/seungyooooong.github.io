공모전 팀원 추천 어플

Team Project, Android, Java

1. 공모전에 함께 참여할 팀원을 추천해주는 앱을 개발한 경험이 있습니다.
2. 프로젝트에서 앱 개발 전체적인 부분을 담당하게 되었는데, 그 당시 장기적으로 프로젝트를 짜고 진행한 경험이 처음이었고 팀 단위로 협업하는 것에 익숙하지 않았습니다.
3. 팀원들의 역량도 제대로 파악하지 못하고 있었으며, 협업할 때 어떻게 업무를 나누어야 하는지도 알지 못했습니다.
4. 그래서 가장 먼저 앱의 여러 가지 기능들을 작은 단위로 모듈화하였고 회의를 통해 팀원들 각각의 능력에 대해서도 꾸준히 조사하였습니다.
5. 그 결과 팀원들의 역량을 파악할 수 있었고 모듈 단위로 팀원들에게 역량에 맞는 업무를 할당하면서 성공적으로 프로젝트를 진행할 수 있었습니다.
6. (drawer layout이나 view pager같은 실전에서 많이 쓰이는 컴포넌트들을 다뤄볼 수 있는 좋은 기회였습니다.)

  • Form: Android App
  • Language: Java

HIG Reading Series

HIG, iOS, velog

1. 애플에서 제공하는 UI 가이드 HIG; Human Interface Guideline를 읽고있습니다.
2. iOS 개발을 꿈꾸면서 여러 자료를 찾아보았는데, 자료 중 가장 도움이 되었던 자료는 실무자가 HIG를 읽으면서 설명해주는 영상 자료였습니다.
3. 해당 영상은 HIG의 내용을 해석해 읽어주고, 실무자가 실제로 개발한 경험을 바탕으로 예시를 들면서, 3-1. (리젝 사유가 될 수 있다.)
3-2. 당시 개발을 하면서 영어 문서를 많이 접하게 된다는 것을 느끼고 더 공부가 필요하겠다고 느끼던 중이었기 때문에 더 기억이 남았던 것 같습니다.
4. iOS 버전업에 따라서 HIG의 내용도 계속 갱신되었고, 나와 같은 경험을 다른 사람들도 할 수 있으면 좋겠다고 생각했습니다.
4-1. 최신화된 HIG를 읽고 내 나름대로 해석하여 직접 자료를 남기고 싶다고 생각했습니다.
4-2. velog에 공부한 내용을 기록하면 좋을 것 같다고 생각했고,
4-2. markdown 언어가 익숙하지 않았지만 크게 어렵지 않아 아주 조금씩이라도 매일 문서를 읽어나가는 시리즈를 만들자고 생각하였습니다.
5. (이후 계획이나 velog라는 플랫폼을 이용해서 얻을 수 있었던 이점 또는 느낀점)
